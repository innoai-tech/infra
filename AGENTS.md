# infra AGENTS

## 文件约定

- 运行 `just` 查看所有可用命令（Go 工具链：`just go ...`，示例服务：`just example ...`）
- 项目背景和模块索引见 `README.md` 和 `docs/`
- 行为约束见本文档

## 开发环境

- Go ≥ 1.26（见 `go.mod`）
- 无需额外初始化，直接 `just go test`、`just go gen` 等入口

## 编码风格

- 文档、注释、错误和测试说明使用中文，专有名词除外
- 先做最小盘点再开始改动
- 不做与任务无关的功能扩展、顺手重构或批量风格清理
- 不因目录已存在而默认新增模块、入口或注册关系
- 不回滚或覆盖自己未引入的变更，除非用户明确要求
- 优先通过参数传递或依赖注入传递上下文，避免可变全局状态
- 保持既有架构分层
- 能完成最小验证时先验证再交付；无法验证时说明缺口与原因

## 什么时候停、什么时候问

- 需求边界不清且存在多种互斥实现时停下
- 需要新增模块、入口或执行面时先确认
- 请求与仓库当前结构或长期维护约束明显冲突时先指出风险
- 关键依赖或外部系统无法在当前工作区内完成验证时先确认
- 需要执行破坏性操作时请求人工处理
- 仓库结构异常，局部 patch 会掩盖真实问题时交用户决策

## 规则生效

- 只有落盘到 `AGENTS.md` 的规则才生效
- 子目录 `AGENTS.md` 只覆盖确有必要的局部规则，未覆盖的继承上级
